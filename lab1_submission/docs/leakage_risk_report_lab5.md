# Leakage Risk Report - Lab 5

## 1. Стратегія split
Обрано **GroupShuffleSplit** за колонкою `canonical_name`. Це запобігає витоку інформації про один і той самий географічний об'єкт між тренувальною та тестовою вибірками.

## 2. Статистика сплітів
- **Train**: 80.0% (106,164 записів)
- **Val**: 10.3% (13,727 записів)
- **Test**: 9.7% (12,821 записів)

## 3. Leakage checks results
- **Exact duplicates (Train ∩ Test)**: 0 (повна ізоляція завдяки групуванню)
- **Near-duplicates**: Мінімальні (переважно короткі омоніми)
- **Template leakage**: Не виявлено службових полів у текстах

## 4. Ризики, що залишились
- Омоніми: різні населені пункти з однаковою назвою можуть потрапляти в різні спліти. Це є природним шумом даних, а не технічним витоком.
