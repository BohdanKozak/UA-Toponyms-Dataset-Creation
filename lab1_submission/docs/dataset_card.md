# Dataset Card: UA Toponyms Normalization Dataset

**Назва проєкту**: Нормалізація українських топонімів (Entity Mapping)
**Задача (A/B/C/D)**: Напрям B (Extraction / Entity Resolution).
**Input / Output**: Input — варіативне написання топоніма (surface form, напр. "Львів", "Lvov", "Lviv", "Lemberg"), Output — канонічна офіційна назва ("Lviv" або "Львів" згідно еталону GeoNames).

**Джерело даних**: Відкрита база GeoNames (dump файлу `UA.zip` - [https://download.geonames.org/export/dump/](https://download.geonames.org/export/dump/)).
**Обсяг**: Понад 30,000+ пар (surface_form -> canonical_name). Більше 15,000 унікальних населених пунктів (класів/сутностей).
**Мова/домени**: Українська, англійська, історичні транслітерації (домен — географія України).

**Очищення**:
- Відфільтровано лише населені пункти (feature_class == 'P').
- Прибрано зайві пробіли.
- Уніфіковано всі типи апострофів ('`‘) до єдиного стандарту (’).
- Використано регулярні вирази для видалення випадкових URL/Email (хоча для гео-даних це рідкість).

**Ризики**:
- *Дисбаланс*: Великі міста мають понад 100 варіантів написання різними мовами світу (арабська, китайська тощо), а дрібні села — лише 1.
- *Шум*: В `alternatenames` можуть потрапляти поштові індекси (цифрові значення) або абревіатури.
- *Упередження*: Можливе домінування російськомовних транслітерацій (Lvov, Kharkov) у старих записах бази порівняно з актуальними (Lviv, Kharkiv).

**План наступного кроку**: У Lab 2 відфільтрувати варіанти написання виключно до кирилиці та латиниці (видалити ієрогліфи), відкинути варіанти коротші за 3 символи та налаштувати пошук нечітких дублікатів.
